# SpaceClaim 스크립트 생성 프롬프트 템플릿 (오프라인, CHM 근거 강제)

목표: SpaceClaim 내부에서 실행 가능한 스크립트를 생성한다. 기능은:

1) 사용자가 지정한 “영역(바디/면/그룹)”의 질량, 관성모멘트(Ixx/Iyy/Izz/Ixy/Iyz/Izx), 질량중심(COG)  
2) 대략적인 형상 특징 추정(실린더: ID/OD, 원판: OD/두께 등)  
3) 결과를 CSV/JSON으로 저장

## 입력(사용자가 채움)

- SpaceClaim 버전: (예: V22)
- 영역 지정 방식:
  - (A) Named selection / 그룹 이름: `...`
  - (B) 선택된 객체(현재 selection): `use_current_selection=true`
  - (C) 좌표 범위로 “잘라내서 선택” (복수 구간 지원)
    - v1은 **1축만 지원** (x 또는 y 또는 z 중 하나)
    - 예: `z=100~250mm`
    - 예: `ranges=[{axis:"z", min:100, max:250}, {axis:"z", min:400, max:520}]`
- 좌표계: global / local (이름)
- 단위: mm, kg 등
- 출력 경로: `C:\\temp\\spaceclaim_props.csv`

## 강제 규율(중요)

1. **CHM 근거 조각 없이 API 호출 금지**
   - SpaceClaim API 객체/메서드/프로퍼티는 반드시 CHM에서 발췌한 근거(토픽 경로/제목)를 함께 제시하고, 코드에도 주석으로 locator를 남긴다.
2. **선택/영역 로직을 먼저 고정**
   - (A)/(B)/(C) 중 **하나만** 선택해 구현한다. 둘 이상 섞지 않는다.
3. **출력 포맷 고정**
   - CSV 헤더를 고정하고(예: `segment_id,mass,cog_x,cog_y,cog_z,Ixx,Iyy,Izz,Ixy,Iyz,Izx,feature_type,feature_params`), 누락값은 빈 칸으로 둔다.

## 출력(에이전트가 생성해야 하는 것)

- SpaceClaim 내부 실행 스크립트 1개
- 스크립트 상단에:
  - 목적/가정(좌표계/단위/영역 지정)
  - 사용한 CHM 근거 토픽 목록(최소 2개)

## 좌표 범위 “잘라내서 선택” 해석(권장 동작)

- 입력이 단일 구간이면 결과는 1개 segment로 출력
- 입력이 복수 구간이면:
  - 각 구간을 `segment_id=1..N`으로 처리
  - 각 segment별로 “그 구간 안에 있는 것”만 대상으로 질량특성/형상특징을 별도로 뽑아 CSV를 여러 줄로 출력
- “잘라내기”의 구현은 두 단계로 나눠서 안정성을 올린다:
  1) **후보 선택**: 바운딩박스가 구간과 교차하는 바디/면을 찾기
  2) **(가능하면) 실제 절단**: 구간 박스(또는 평면)로 바디를 split/trim하여 구간 내부만 남기기
     - CHM 근거에서 안전한 API가 확인될 때만 수행하고, 불확실하면 1)까지만 수행

## 관성모멘트 기준점(중요)

출력하는 관성모멘트 텐서는 반드시 **각 segment의 질량중심(COG) 기준**이어야 한다.
- CHM 근거에서 “COG 기준 관성”을 직접 제공하면 그 값을 사용
- 그렇지 않으면:
  - (A) SpaceClaim이 주는 관성이 원점 기준인지/COG 기준인지 먼저 확인하고(근거 포함)
  - (B) 원점 기준만 있다면 병진축 정리(Parallel Axis Theorem)로 COG 기준으로 변환
    - 이 경우 변환식/가정을 코드 주석에 남긴다(근거 + 수학식)

## 힌트: “형상 특징”은 1차는 단순하게

- 원판(disk) 후보: 두께 방향 치수 ≪ 나머지 두 축 치수
- 실린더 후보: 두 축 치수가 비슷하고, 한 축이 길거나(파이프) 두께가 일정

형상 특징은 v1에서 과하지 않게:
- “type”만 제대로 맞추고, 치수(ID/OD/두께)는 “가능하면” 계산
- 근거 없는 계산/가정은 하지 말고, **추정**이면 출력에 `estimated=true`를 남긴다
