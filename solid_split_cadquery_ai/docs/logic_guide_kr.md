# 시스템 로직 상세 가이드

본 프로젝트의 핵심 3단계 로직에 대한 상세 기술 가이드입니다.

## 1. 형상 데이터 추출 (Geometry Extraction)
AI가 형상을 올바르게 판단할 수 있도록, 기하학적 정보를 **추상화**하여 전달합니다.

*   **물리적 면(Face) -> 논리적 피처(Feature)**:
    *   모든 `Cylinder` 타입 면을 전수 조사합니다.
    *   면의 법선(Normal)이 중심축을 향하면 `hole`, 밖을 향하면 `boss`로 자동 분류합니다.
    *   구멍들의 위치와 반지름을 비교하여 패턴(Pattern)을 형성합니다. 이는 AI가 "이 구멍들은 한꺼번에 처리해야 할 격자 패턴이다"라고 인식하게 돕습니다.
*   **좌표계 요약**:
    *   전체 Bounding Box를 통해 절대적인 공간 크기를 제공합니다.
    *   Z축 방향의 평면들을 분석하여 단차(Step)의 높이와 위치를 목록화합니다.

## 2. AI 응답 프로토콜 (AI Response Protocol)
AI의 자유로운 답변을 억제하고, 코드에서 즉시 파싱 가능한 **Strict JSON** 형식을 사용합니다.

*   **명령 스키마**:
    *   `plane_cut`: 특정 좌표에서 면을 자르는 명령. (예: `{"axis": "Z", "coordinate": 15.0}`)
    *   `hole_isolation`: 특정 구멍을 격리하는 복합 명령. (예: `{"center": [25, 25, 0], "radius": 5.0}`)
*   **프롬프트 엔지니어링**:
    *   AI에게 "격자 품질(Hex-Mesh Quality)"을 최우선 목표로 제시합니다.
    *   단순히 자르는 것이 아니라, 잘린 이후의 각 조각이 육면체에 가까워져야 함을 강조합니다.

## 3. 기하 연산 실행 (Geometric Execution)
AI가 설계한 "도면"을 CadQuery를 통해 "실체화"합니다.

*   **Workplane 기반 절단**:
    *   대상 바디의 중심을 통과하는 임시 평면을 생성하고 `split()` 연산을 수행합니다.
    *   `keepTop=True, keepBottom=True` 옵션을 통해 잘린 양쪽 조각을 모두 보존하며 관리합니다.
*   **재귀적 관리**:
    *   한 번 절단이 일어날 때마다 관리 대상 바디 리스트를 업데이트하여, 이후의 절단 명령이 파편화된 모든 조각에 유효하게 적용되도록 합니다.
